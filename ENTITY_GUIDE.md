# Инструкция по добавлению новых типов сущностей

Чтобы добавить новую категорию (например, `bosses` или `quests`), выполните следующие шаги:

### 1. Определите схему данных
В файле `src/shared/types/entities.ts`:
1. Добавьте название категории в перечисление `EntityCategory`.
2. Создайте Zod-схему для новой сущности (используйте `.extend()` от `BaseEntitySchema`).
3. Добавьте новую схему в объединение `Entity`.

### 2. Подключите валидацию
В файле `src/shared/api/dataService.ts`:
1. Импортируйте новую схему.
2. Добавьте её в `SchemaMap`, сопоставив ключ категории со схемой.

### 3. Создайте файл данных
1. В папке `public/data/` создайте пустой файл `название_категории.json` (например, `bosses.json`).
2. В файле `public/data/registry.json` добавьте имя этого файла в массив.

### 4. Добавьте переводы
В файле `src/shared/lib/i18n.ts`:
1. В секции `en.translation.categories` и `ru.translation.categories` добавьте название вашей категории для отображения в интерфейсе.

### 5. Обновите интерфейс (опционально)
1. На странице `Home.tsx` в массиве `categories` добавьте новую категорию, чтобы она появилась в списке на главной.
2. В `AdminPanel.tsx` в списке категорий добавьте новую строку, чтобы она была доступна в редакторе.
3. Если у сущности есть уникальные поля, добавьте их отображение в `EntityDetail.tsx` и `AdminPanel.tsx`.
